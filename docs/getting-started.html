Getting Started

Prerequisites

Before you begin, ensure you have the following installed:

Software Version Purpose
Java JDK 11 or higher Core runtime environment
Apache Maven 3.6+ Build and dependency management
Git Latest Version control
Chrome/Firefox Latest Web browser for testing
IDE IntelliJ/Eclipse Development environment
Note: Selenium Manager will automatically download browser drivers. No manual driver setup required!

Installation Steps

Step 1: Clone the Repository

git clone https://github.com/YOUR_USERNAME/SandsAutoFramework.git
cd SandsAutoFramework

Step 2: Verify Java Installation

java -version
# Should show Java 11 or higher

mvn -version
# Should show Maven 3.6 or higher

Step 3: Build the Project

mvn clean install

This command will:

  • Download all dependencies
  • Compile source code
  • Run unit tests (if any)
  • Package the framework as JAR

Step 4: Install Playwright Browsers (Optional)

If you plan to use Playwright:

mvn exec:java -e -D exec.mainClass=com.microsoft.playwright.CLI -D exec.args="install"

IDE Setup

IntelliJ IDEA

  1. Open IntelliJ IDEA
  2. File → Open → Select SandsAutoFramework folder
  3. Wait for Maven to import dependencies
  4. Install Cucumber plugin: File → Settings → Plugins → Search "Cucumber for Java"
  5. Enable annotation processing: Settings → Build → Compiler → Annotation Processors

Eclipse

  1. Open Eclipse
  2. File → Import → Maven → Existing Maven Projects
  3. Select SandsAutoFramework folder
  4. Install Cucumber plugin from Eclipse Marketplace

VS Code

  1. Open VS Code
  2. File → Open Folder → Select SandsAutoFramework
  3. Install extensions: Java Extension Pack, Cucumber (Gherkin)

Configuration

Basic Configuration

Edit src/main/resources/config.properties:

# Framework Type: selenium | playwright | api | mobile | desktop
framework.type=selenium

# Browser: chrome | firefox | edge | safari
browser=chrome

# Headless Mode
headless=false

# Environment: qa | uat | staging | prod
environment=qa

# Parallel Execution
parallel.execution=true
thread.count=3

Environment-Specific Configuration

Create environment files in src/main/resources/:

qa.properties

app.url=https://qa.example.com
api.base.url=https://api-qa.example.com
db.url=jdbc:mysql://qa-db:3306/testdb

prod.properties

app.url=https://example.com
api.base.url=https://api.example.com
db.url=jdbc:mysql://prod-db:3306/proddb

Running Your First Test

Run All Tests

mvn clean test

Run Specific Tags

# Run UI tests only
mvn test -Dcucumber.filter.tags="@UI"

# Run API tests only
mvn test -Dcucumber.filter.tags="@API"

# Run Smoke tests
mvn test -Dcucumber.filter.tags="@Smoke"

# Run multiple tags
mvn test -Dcucumber.filter.tags="@UI and @Smoke"

Run with Different Browser

mvn test -Dbrowser=firefox
mvn test -Dbrowser=edge

Run in Headless Mode

mvn test -Dheadless=true

Run with Different Environment

mvn test -Denvironment=staging

Run Specific Feature File

mvn test -Dcucumber.features="src/test/resources/features/login.feature"

Creating Your First Test

1. Create Feature File

Create src/test/resources/features/login.feature:

@UI @Smoke
Feature: Login Functionality

  Scenario: Successful login with valid credentials
    Given user opens the login page
    When user enters username "testuser" and password "Test@123"
    And user clicks on login button
    Then user should see the dashboard
    And user should see welcome message "Welcome, Test User"

2. Create Page Object

Create src/main/java/com/automation/pages/LoginPage.java:

package com.automation.pages;

import com.automation.reusables.SeleniumReusable;
import org.openqa.selenium.By;

public class LoginPage extends SeleniumReusable {
    
    // Locators
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton = By.id("loginBtn");
    private By welcomeMessage = By.className("welcome-msg");
    
    // Methods
    public void openLoginPage() {
        navigateTo(ConfigManager.getProperty("app.url") + "/login");
    }
    
    public void enterUsername(String username) {
        type(usernameField, username);
    }
    
    public void enterPassword(String password) {
        type(passwordField, password);
    }
    
    public void clickLogin() {
        click(loginButton);
    }
    
    public String getWelcomeMessage() {
        return getText(welcomeMessage);
    }
}

3. Create Step Definitions

Create src/test/java/com/automation/stepdefinitions/LoginSteps.java:

package com.automation.stepdefinitions;

import com.automation.pages.LoginPage;
import com.automation.core.assertions.AssertUtils;
import io.cucumber.java.en.*;

public class LoginSteps {
    
    private LoginPage loginPage = new LoginPage();
    
    @Given("user opens the login page")
    public void userOpensLoginPage() {
        loginPage.openLoginPage();
    }
    
    @When("user enters username {string} and password {string}")
    public void userEntersCredentials(String username, String password) {
        loginPage.enterUsername(username);
        loginPage.enterPassword(password);
    }
    
    @When("user clicks on login button")
    public void userClicksLoginButton() {
        loginPage.clickLogin();
    }
    
    @Then("user should see the dashboard")
    public void userShouldSeeDashboard() {
        String currentUrl = DriverManager.getSeleniumDriver().getCurrentUrl();
        AssertUtils.assertTrue(currentUrl.contains("/dashboard"), 
            "User should be on dashboard page");
    }
    
    @Then("user should see welcome message {string}")
    public void userShouldSeeWelcomeMessage(String expectedMessage) {
        String actualMessage = loginPage.getWelcomeMessage();
        AssertUtils.assertEquals(actualMessage, expectedMessage, 
            "Welcome message should match");
    }
}

4. Run Your Test

mvn test -Dcucumber.filter.tags="@Smoke"

Troubleshooting

Issue: Maven dependencies not downloading

Solution: 
mvn clean install -U
# -U forces update of dependencies

Issue: Browser driver not found

Solution: Selenium Manager auto-downloads drivers. Ensure you have internet connection and Chrome/Firefox installed.

Issue: Tests failing in parallel

Solution: Reduce thread count in config.properties: 
thread.count=2

Issue: Playwright browsers not installed

Solution: 
mvn exec:java -e -D exec.mainClass=com.microsoft.playwright.CLI -D exec.args="install"

Issue: Port already in use (Appium/WinAppDriver)

Solution: Kill the process using the port: 
# macOS/Linux
lsof -ti:4723 | xargs kill -9

# Windows
netstat -ano | findstr :4723
taskkill /PID [PID_NUMBER] /F
