Reporting & Logging

Overview

Unified reporting system that logs to multiple reporting tools simultaneously.

Reporting Tools

  • ExtentReports - Modern HTML reports with dark theme
  • CustomReporter - Lightweight HTML with embedded screenshots
  • Allure - Interactive dashboard with trends
  • Cucumber HTML - Standard Cucumber reports
  • LogManager - File-based logging

UnifiedLogger

Single logging call logs to all reporting systems simultaneously.

Usage

// Info logging
UnifiedLogger.info("Test started");
UnifiedLogger.info("Navigating to login page");

// Pass/Fail logging
UnifiedLogger.pass("Login successful");
UnifiedLogger.fail("Expected element not found");

// Action logging (for reusable methods)
UnifiedLogger.action("Click", loginButton);
UnifiedLogger.action("Type", "username field | testuser");

// Warning logging
UnifiedLogger.warn("Element took longer than expected");

// Error with exception
UnifiedLogger.error("API call failed", exception);

// Debug logging (file only)
UnifiedLogger.debug("Debug information");

ExtentReports

Modern HTML reports with dark theme and rich features.

Features

  • Dark theme with modern UI
  • Screenshots embedded in report
  • Category and author filtering
  • System information display
  • Timeline view
  • Pass/Fail statistics

Direct API (Optional)

// Usually not needed - use UnifiedLogger instead
ExtentReporter.startTest("Test Name");
ExtentReporter.logInfo("Info message");
ExtentReporter.logPass("Pass message");
ExtentReporter.logFail("Fail message");
ExtentReporter.attachScreenshot(screenshotPath);
ExtentReporter.assignCategory("Smoke", "UI");
ExtentReporter.assignAuthor("Afsar Ali");
ExtentReporter.endTest();

CustomReporter

Lightweight HTML reporter with embedded screenshots as base64.

Features

  • Single HTML file (portable)
  • Screenshots embedded as base64
  • Pass/Fail summary
  • Test duration tracking
  • Detailed step logs
  • No external dependencies

Direct API (Optional)

// Usually not needed - use UnifiedLogger instead
CustomReporter.startTest("Test Name");
CustomReporter.logInfo("Info message");
CustomReporter.logPass("Pass message");
CustomReporter.logFail("Fail message");
CustomReporter.attachScreenshot(screenshotPath);
CustomReporter.endTest();

Allure Reports

Interactive dashboard with trends and detailed analytics.

Features

  • Interactive dashboard
  • Timeline view
  • Trend analysis
  • Test history
  • Categories and suites
  • Attachments support

Generate Allure Report

# Generate and open report
mvn allure:serve

# Generate report only
mvn allure:report

# Report location
target/site/allure-maven-plugin/index.html

Allure Annotations

import io.qameta.allure.*;

@Epic("E-Commerce")
@Feature("Login")
@Story("User Authentication")
@Severity(SeverityLevel.CRITICAL)
@Test
public void testLogin() {
    Allure.step("Navigate to login page");
    Allure.step("Enter credentials");
    Allure.step("Click login button");
    
    // Attach screenshot
    Allure.addAttachment("Screenshot", "image/png", 
                         new ByteArrayInputStream(screenshot), "png");
}

File Logging

Detailed file-based logging using SLF4J and Logback.

Log Levels

Level Usage
DEBUG Detailed debugging information
INFO General informational messages
WARN Warning messages
ERROR Error messages with exceptions

Log Location

test-output/logs/automation.log

Log Configuration (logback.xml)

<configuration>
    <appender name="FILE" class="ch.qos.logback.core.FileAppender">
        <file>test-output/logs/automation.log</file>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <root level="INFO">
        <appender-ref ref="FILE" />
    </root>
</configuration>

Best Practices

  • Always use UnifiedLogger for consistent logging
  • Use meaningful log messages
  • Log actions, not just results
  • Attach screenshots on failures
  • Use appropriate log levels
  • Review all report types for different insights
  • Archive reports for historical analysis
  • Use Allure for trend analysis

Example Test with Logging

@Test
public void testLogin() {
    UnifiedLogger.info("Starting login test");
    
    UnifiedLogger.action("Navigate", appUrl);
    driver.get(appUrl);
    
    UnifiedLogger.action("Type", "username field");
    loginPage.enterUsername("testuser");
    
    UnifiedLogger.action("Type", "password field");
    loginPage.enterPassword("password123");
    
    UnifiedLogger.action("Click", "login button");
    loginPage.clickLogin();
    
    if (dashboardPage.isDisplayed()) {
        UnifiedLogger.pass("Login successful - Dashboard displayed");
    } else {
        UnifiedLogger.fail("Login failed - Dashboard not displayed");
    }
}
