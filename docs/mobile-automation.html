Mobile Automation (Appium)

Overview

Mobile automation using Appium for Android and iOS testing with W3C Actions API.

Key Features

  • Android and iOS support
  • W3C Actions API for gestures
  • Platform-specific methods
  • Touch gestures (swipe, tap, long press)
  • Native app automation

Configuration

# config.properties
framework.type=mobile

# Android Configuration
mobile.platformName=Android
mobile.deviceName=emulator-5554
mobile.appiumServerUrl=http://127.0.0.1:4723
mobile.appPackage=com.example.android
mobile.appActivity=com.example.android.MainActivity
mobile.automationName=UiAutomator2

# iOS Configuration
mobile.platformName=iOS
mobile.deviceName=iPhone 14
mobile.udid=auto
mobile.bundleId=com.example.ios
mobile.automationName=XCUITest

MobileReusable Methods

App Lifecycle

Method Parameters Description
launchApp() URL, DesiredCapabilities, boolean Launch Android/iOS app
closeApp() - Close app and quit driver

Element Interaction

Method Parameters Description
click() By locator Click element
type() By locator, String text Clear and type text
getText() By locator Get element text
findElement() By locator Find single element
findElements() By locator Find multiple elements

Touch Gestures (W3C Actions)

Method Parameters Description
swipe() int startX, startY, endX, endY, durationMs Swipe from one point to another
tap() int x, y Tap at coordinates
longPress() int x, y, int durationMs Long press at coordinates
dragAndDrop() By source, By target Drag element to target

Page Object Example

public class LoginPage extends MobileReusable {
    
    // Locators
    By usernameField = By.id("com.example:id/username");
    By passwordField = By.id("com.example:id/password");
    By loginButton = By.id("com.example:id/login");
    By errorMessage = By.id("com.example:id/error");
    
    public void login(String username, String password) {
        type(usernameField, username);
        type(passwordField, password);
        click(loginButton);
        UnifiedLogger.action("Login", username);
    }
    
    public boolean isErrorDisplayed() {
        return findElement(errorMessage).isDisplayed();
    }
    
    public void swipeToBottom() {
        swipe(500, 1500, 500, 500, 1000);
    }
}

Step Definitions

public class MobileSteps extends MobileReusable {
    
    @Given("mobile app is launched")
    public void mobileAppIsLaunched() {
        // App launched via hooks
        UnifiedLogger.info("Mobile app launched");
    }
    
    @When("user enters username {string}")
    public void userEntersUsername(String username) {
        type(By.id("username"), username);
    }
    
    @When("user swipes down on screen")
    public void userSwipesDown() {
        swipe(500, 500, 500, 1500, 1000);
    }
    
    @When("user taps at coordinates {int} {int}")
    public void userTapsAt(int x, int y) {
        tap(x, y);
    }
}

Locator Strategies

Android

// Resource ID
By.id("com.example:id/button")

// Accessibility ID
MobileBy.AccessibilityId("Login Button")

// XPath
By.xpath("//android.widget.Button[@text='Login']")

// UiAutomator
MobileBy.AndroidUIAutomator("new UiSelector().text(\"Login\")")

iOS

// Accessibility ID
MobileBy.AccessibilityId("loginButton")

// iOS Predicate
MobileBy.iOSNsPredicateString("name == 'Login'")

// iOS Class Chain
MobileBy.iOSClassChain("**/XCUIElementTypeButton[`name == 'Login'`]")

// XPath
By.xpath("//XCUIElementTypeButton[@name='Login']")

Running Mobile Tests

# Start Appium server
appium

# Run Android tests
mvn test -Dcucumber.filter.tags="@Mobile and @Android"

# Run iOS tests
mvn test -Dcucumber.filter.tags="@Mobile and @iOS"

# Run specific feature
mvn test -Dcucumber.options="--features src/test/resources/features/mobile_login.feature"

Best Practices

  • Use resource IDs for Android (most stable)
  • Use Accessibility IDs for iOS
  • Avoid XPath when possible (slow)
  • Use W3C Actions for gestures
  • Handle platform differences in Page Objects
  • Use explicit waits for dynamic content
  • Test on real devices when possible
