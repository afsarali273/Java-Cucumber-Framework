Overview

Run tests on cloud platforms without local infrastructure. Supports LambdaTest for web and AWS Device Farm for mobile.

Zero Code Changes: Switch between local and cloud execution by updating config.properties only.

Quick Start

1. Local Execution (Default)

# config.properties
remote.provider=
# OR
remote.provider=none
mvn clean test

2. LambdaTest (Web)

# config.properties
remote.provider=lambdatest
lambdatest.username=your_username
lambdatest.accessKey=your_access_key
mvn clean test -Dcucumber.filter.tags="@UI"

3. AWS Device Farm (Mobile)

# config.properties
remote.provider=aws
aws.devicefarm.url=https://appium-us-west-2.devicefarm.aws.amazon.com/wd/hub
aws.devicefarm.app=arn:aws:devicefarm:us-west-2:123456789012:upload:xxxxx
mvn clean test -Dcucumber.filter.tags="@Mobile"

LambdaTest Setup

Step 1: Get Credentials

  1. Sign up at LambdaTest
  2. Go to Profile ‚Üí Account Settings ‚Üí Password & Security
  3. Copy your Username and Access Key

Step 2: Configure Properties

# Enable LambdaTest
remote.provider=lambdatest

# Credentials
lambdatest.username=your_username
lambdatest.accessKey=your_access_key

# Browser & Platform
browser=chrome
lambdatest.browserVersion=latest
lambdatest.platform=Windows 10

# Test Metadata
lambdatest.build=Sands Framework Build
lambdatest.project=Sands Automation

# Features
lambdatest.video=true
lambdatest.visual=false
lambdatest.network=false
lambdatest.console=false

Step 3: Run Tests

# Run all UI tests
mvn clean test -Dcucumber.filter.tags="@UI"

# Override browser
mvn test -Dbrowser=firefox

# Parallel execution
mvn test -Dparallel.execution=true -Dthread.count=5

Step 4: View Results

Dashboard: https://automation.lambdatest.com/

  • View videos, screenshots, logs
  • Network logs and console logs
  • Download test artifacts

AWS Device Farm Setup

Step 1: Upload App

  1. Go to AWS Device Farm Console
  2. Create a new project
  3. Upload your app (.apk for Android, .ipa for iOS)
  4. Note the app ARN

Step 2: Create Test Run

  1. Create a new test run
  2. Select Appium Java TestNG as framework
  3. Select devices
  4. Copy the Appium endpoint URL

Step 3: Configure Properties

# Enable AWS Device Farm
remote.provider=aws

# Framework type
framework.type=mobile

# Device Farm URL
aws.devicefarm.url=https://appium-us-west-2.devicefarm.aws.amazon.com/wd/hub

# Platform
mobile.platformName=Android

# Device & App
aws.devicefarm.deviceName=Samsung Galaxy S21
aws.devicefarm.app=arn:aws:devicefarm:us-west-2:123456789012:upload:xxxxx

# Android specific
mobile.appPackage=com.example.android
mobile.appActivity=com.example.android.MainActivity

Step 4: Run Tests

mvn clean test -Dcucumber.filter.tags="@Mobile"

Step 5: View Results

Dashboard: AWS Device Farm Console

  • View videos and screenshots
  • Device logs and performance metrics
  • Download test artifacts

Parallel Execution

Both LambdaTest and AWS Device Farm support parallel execution:

# Enable parallel execution
parallel.execution=true
thread.count=5

# Each thread gets its own remote session
remote.provider=lambdatest
mvn test -Dparallel.execution=true -Dthread.count=5
Thread-Safe: Each thread creates an isolated remote session. No cross-thread interference.

Helper Scripts

Quickly switch between execution modes:

macOS/Linux

# Switch to local
./switch-execution-mode.sh local

# Switch to LambdaTest
./switch-execution-mode.sh lambdatest

# Switch to AWS Device Farm
./switch-execution-mode.sh aws

# Check current mode
./switch-execution-mode.sh status

Windows

# Switch to local
switch-execution-mode.bat local

# Switch to LambdaTest
switch-execution-mode.bat lambdatest

# Switch to AWS Device Farm
switch-execution-mode.bat aws

Troubleshooting

Tests still running locally

Solution: Check remote.provider is set correctly (not empty, not "none")

Authentication failed (LambdaTest)

Solution: Verify username and access key from LambdaTest dashboard

Connection timeout (AWS Device Farm)

Solution: Ensure Device Farm URL is from active test run

Parallel execution not working

Solution: Verify parallel.execution=true and thread.count is set

Best Practices

  • Use environment variables for credentials
  • Tag tests appropriately (@UI, @Mobile)
  • Set meaningful test names
  • Enable video only for failures
  • Use build numbers for tracking
  • Test with single thread first
  • Monitor cloud usage and costs
